Out of the box a Redis backed cache is implemented.

class RedisCacheDatabase ( TraderId trader_id , Logger logger , Serializer serializer , config: Optional[CacheDatabaseConfig] = None ) ¬∂

Bases: CacheDatabase

Provides a cache database backed by Redis.

Parameters :

  • trader_id ( TraderId ) ‚Äì The trader ID for the database.

  • logger ( Logger ) ‚Äì The logger for the database.

  • serializer ( Serializer ) ‚Äì The serializer for database operations.

  • config ( CacheDatabaseConfig , optional ) ‚Äì The configuration for the instance.

Raises :

TypeError ‚Äì If config is not of type CacheDatabaseConfig .

Warning

Redis can only accurately store int64 types to 17 digits of precision. Therefore nanosecond timestamp int64‚Äôs with 19 digits will lose 2 digits of precision when persisted. One way to solve this is to ensure the serializer converts timestamp int64‚Äôs to strings on the way into Redis, and converts timestamp strings back to int64‚Äôs on the way out. One way to achieve this is to set the timestamps_as_str flag to true for the MsgPackSerializer , as per the default implementations for both TradingNode and BacktestEngine .
